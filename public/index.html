Model Schema Viewer

Schema Format

The schema uses JSON Schema format. Each field can have the following attributes:

Field Attributes

  • type - The data type of the field (string, integer, float, boolean, json, datetime)
  • length - Maximum string length in sqlite
  • unique - Determines if the field has unique values in sqlite
  • primary - Determines if the field is a primary key in sqlite
  • notnull - Determines if the field can hold null values in sqlite

Example Schema

{
  "name": {
    "type": "string",
    "length": 100,
    "primary": true
  },
  "age": {
    "type": "integer"
  },
  "isActive": {
    "type": "boolean"
  }
}

Note: The schema is stored as JSON in the database and will be validated when saved.
